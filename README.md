# LeetCode Database Practice

A comprehensive collection of **LeetCode SQL database problems** and their solutions, organized by difficulty level and problem type.

## 📚 Repository Overview

This repository contains solutions to LeetCode SQL/Database practice problems with detailed explanations and approaches. Perfect for anyone preparing for SQL interviews or looking to strengthen their database skills.

| # | Problem | Difficulty | Solution |
|---|---------|-----------|----------|
| 175 | Combine Two Tables | Easy | SQL JOIN |

## 🚀 Getting Started

1. Each problem is organized in its own file with a clear naming convention: `[Problem_Number]. [Problem_Name]`
2. Each file contains:
   - **Problem Statement** - Clear description of what needs to be solved
   - **Approach** - Explanation of the solution strategy
   - **SQL Solution** - Complete, working SQL code
   - **Key Concepts** - Important SQL concepts and keywords used
   - **Complexity Analysis** - Time and space considerations

## 🛠️ Topics Covered

- **SELECT Statements** - Basic and advanced query writing
- **JOINs** - INNER, LEFT, RIGHT, FULL OUTER joins
- **Window Functions** - ROW_NUMBER(), RANK(), DENSE_RANK(), etc.
- **Subqueries** - Nested queries and derived tables
- **Aggregation** - GROUP BY, HAVING, COUNT(), MAX(), MIN(), AVG()
- **String Functions** - CONCAT(), SUBSTRING(), UPPER(), LOWER()
- **Date Functions** - DATE manipulation and formatting
- **Common Table Expressions (CTEs)** - WITH clauses

## 📖 How to Use This Repository

1. **Practice** - Try solving each problem before looking at the solution
2. **Learn** - Read through the approach and key concepts
3. **Review** - Compare your solution with the provided one
4. **Optimize** - Look for better approaches and understand trade-offs

## 💡 Tips for SQL Interview Preparation

- ✅ Write clean, readable queries with proper indentation
- ✅ Use meaningful aliases for tables and columns
- ✅ Test edge cases (NULL values, empty result sets, duplicates)
- ✅ Consider performance implications for large datasets
- ✅ Practice explaining your approach verbally
